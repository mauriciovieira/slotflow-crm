# Resume Models — Design

**Date:** 2026-04-25
**Status:** Approved
**Scope:** Track 03 next entity. `BaseResume` + `ResumeVersion`. Backend-only — model + migration + admin + per-app tests. **No DRF API in this PR**, parallel to the PR F → PR G split that worked well for Opportunity.

## Goal

Land the resume scaffolding the import + render pipeline (Track 05) will need. Two models: a workspace-owned `BaseResume` (the "canonical" resume), and an append-only `ResumeVersion` snapshot (the JSON Resume document at a point in time). Sections / items / evidence map come in later PRs.

## Non-goals

- DRF API for resumes — separate later PR (parallel to opportunities PR G).
- `ResumeSection`, `ResumeItem`, `ResumeEvidenceMap` — later sub-scope under Track 05.
- `ResumeImportJob`, `ResumeRender` (job/render pipeline) — later Track 05 PR.
- JSON Resume schema validation. The document is stored as `JSONField` and trusted; validation lives in the importer.
- Theme rendering, file storage abstractions.
- Frontend / e2e changes.

## Architecture

### Models (`backend/resumes/models.py`)

```py
class BaseResume(TimeStampedModel):
    id = UUIDField(primary_key, default=uuid4, editable=False)
    workspace = FK("tenancy.Workspace", on_delete=CASCADE,
                   related_name="resumes")
    name = CharField(max_length=200)             # "Senior Eng — backend focus"
    created_by = FK(User, on_delete=SET_NULL, null=True, blank=True,
                    related_name="created_resumes")

    class Meta:
        ordering = ("-created_at",)
        indexes = [Index(fields=("workspace",))]


class ResumeVersion(TimeStampedModel):
    id = UUIDField(primary_key, default=uuid4, editable=False)
    base_resume = FK(BaseResume, on_delete=CASCADE, related_name="versions")
    version_number = PositiveIntegerField()      # 1-indexed, scoped per base_resume
    document = JSONField()                       # canonical JSON Resume snapshot
    document_hash = CharField(max_length=64)     # sha256 of canonical-serialized document
    notes = TextField(blank=True, default="")
    created_by = FK(User, on_delete=SET_NULL, null=True, blank=True,
                    related_name="created_resume_versions")

    class Meta:
        ordering = ("-version_number",)
        constraints = [
            UniqueConstraint(fields=("base_resume", "version_number"),
                             name="uniq_resume_version_number"),
        ]
        indexes = [Index(fields=("base_resume", "version_number"))]
```

Choices:

- **UUID pk** matches Workspace, Membership, Opportunity, McpToken.
- **`BaseResume` cascades on workspace delete.** Same shape as Opportunity. If the workspace goes, its resumes go too.
- **`created_by` SET_NULL** on user delete on both models — keeps the resume artifact even after the author leaves the workspace.
- **`ResumeVersion` cascades on base resume delete.** Versions are sub-resources of the base; orphaned versions make no sense.
- **`version_number` is workspace-of-base scoped by the unique constraint.** Each base has its own monotonically-increasing version sequence (1, 2, 3 …). Computed at write time by services in a later PR; this PR only enforces uniqueness.
- **`document_hash` exists for future deduplication.** Track 05 importer can skip a no-op import by comparing hashes. Computed at write time by services later. NOT made unique — same content might intentionally appear under two bases (forking a CV).
- **No `is_current` flag.** Latest by `version_number` is the canonical "current" snapshot. Adding a flag invites two-source-of-truth bugs.
- **`document` as `JSONField`.** Postgres backs this as JSONB; SQLite (test runner) treats it as TEXT. Either is fine — we don't query into the JSON yet.

### Migration

`backend/resumes/migrations/0001_initial.py`, autogenerated. Two `CreateModel` ops + one `AddConstraint` for the version uniqueness. Default `make migrate` picks it up; no Makefile change.

### Admin (`backend/resumes/admin.py`)

Register both with sensible inlines:

- `BaseResumeAdmin`: `list_display = ("name", "workspace", "created_by", "created_at")`, `list_filter = ("workspace",)`, `search_fields = ("name",)`, `readonly_fields = ("id", "created_at", "updated_at")`. A `TabularInline` for `ResumeVersion` (`readonly_fields = ("version_number", "document_hash", "created_at")`, `extra = 0`) so the admin can inspect versions inline without editing them.
- `ResumeVersionAdmin`: `list_display = ("base_resume", "version_number", "document_hash", "created_at")`, `list_filter = ("base_resume__workspace",)`, `readonly_fields = ("id", "version_number", "document_hash", "created_at", "updated_at")`. Document JSON stays editable in the form for now (admin convenience); the API path will treat it as immutable when it lands.

### Tests (per-category layout)

`backend/resumes/tests/models/`:

- `base_resume_test.py` — seven cases:
  1. Minimum-field create + defaults.
  2. `__str__` formats as `f"{name} (workspace.slug)"`.
  3. Workspace cascade.
  4. Creator delete nullifies `created_by`.
  5. Default ordering newest-first.
  6. Two BaseResumes in different workspaces don't conflict on name.
  7. The `versions` reverse accessor returns the related `ResumeVersion` rows.

- `resume_version_test.py` — eight cases:
  1. Minimum-field create with `version_number=1`, asserts `document` round-trips a small dict.
  2. `(base_resume, version_number)` unique — second create with same pair raises `IntegrityError`.
  3. Different bases can each have their own `version_number=1`.
  4. Cascade on base resume delete.
  5. Default ordering inside one base is newest-version first.
  6. Creator delete nullifies `created_by`.
  7. `__str__` formats as `f"{base.pk} v{version_number}"`.
  8. `version_number=0` is rejected by the `check_constraint`.

Fifteen new tests, 111 → 126.
