# Multi-package changelog implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement independent backend and frontend semantic releases with English changelogs under each package, prefixed Git tags, path-scoped commit analysis, and a root `CHANGELOG.md` chronological index updated after each release.

**Architecture:** Configure `python-semantic-release` in `backend/` with the `conventional-monorepo` parser, `backend-v{version}` tags, and `backend/CHANGELOG.md`. Add `semantic-release` (npm) under `frontend/` with `semantic-release-commit-filter` (run from `frontend/`), `frontend-v{version}` tags, `frontend/package.json` version starting at `0.0.1`, and `frontend/CHANGELOG.md`. Split GitHub Actions so each release job runs on relevant path changes; after a successful release, prepend one index entry to the root `CHANGELOG.md` (with `git pull --rebase` before push). Migrate legacy unprefixed tags by adding an initial `backend-v*` tag aligned to the current backend version.

**Tech Stack:** GitHub Actions, python-semantic-release v10.x, semantic-release (npm), semantic-release-commit-filter, Conventional Commits, bash.

| File | Role |

| `.github/workflows/release-frontend.yml` | Path-filtered push to `main`, Node, `semantic-release`, root prepend, push |
| Remove or replace `.github/workflows/release.yml` | Avoid duplicate releases |

Use this exact `[tool.semantic_release]` block (merge with existing `[project]`, `[tool.ruff]`, `[tool.pytest.ini_options]` — do not remove those sections):

```toml
[tool.semantic_release]
allow_zero_version = true
commit_parser = "conventional-monorepo"
commit_message = """chore(release): backend {version} [skip ci]

Automatically generated by python-semantic-release"""
tag_format = "backend-v{version}"
version_toml = ["pyproject.toml:project.version"]
version_variables = ["config/version.py:__version__"]

[tool.semantic_release.commit_parser_options]
path_filters = ["."]
scope_prefix = "^backend"

[tool.semantic_release.changelog.default_templates]
changelog_file = "CHANGELOG.md"
```

Notes:
- `path_filters = ["."]` is relative to the **package directory** (`backend/` when PSR runs with `directory: backend`). See [PSR monorepo guide](https://python-semantic-release.readthedocs.io/en/latest/configuration/configuration-guides/monorepos.html).

### Task 4: Root `CHANGELOG.md` template and prepend script

**Files:**
- Modify: `CHANGELOG.md` (repository root)
- Create: `scripts/prepend-root-changelog.sh`

- [ ] **Step 1: Replace root `CHANGELOG.md`** with an English index template (adjust repo name in intro if needed):

```markdown
# Changelog

This file is a **chronological index** of package releases (newest first). Detailed notes live in:

- [Backend](backend/CHANGELOG.md)
- [Frontend](frontend/CHANGELOG.md)

<!-- release-index -->
```

The `<!-- release-index -->` marker is optional; the script below prepends **after** the first heading block. If you prefer no marker, prepend after line 7 (first blank line after intro).

- [ ] **Step 2: Create `scripts/prepend-root-changelog.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail

# Usage: scripts/prepend-root-changelog.sh <backend|frontend> <x.y.z>
# Example: scripts/prepend-root-changelog.sh backend 0.3.0

PKG="${1:?package: backend or frontend}"
VER="${2:?version x.y.z}"
DATE="$(date -u +%Y-%m-%d)"
ROOT="$(git rev-parse --show-toplevel)"
FILE="${ROOT}/CHANGELOG.md"

case "$PKG" in
  backend) SUB="Backend"; REL="backend" ;;
  frontend) SUB="Frontend"; REL="frontend" ;;
  *) echo "package must be backend or frontend" >&2; exit 1 ;;
esac

# Link to package changelog; fragment matches GitHub slug for ## x.y.z style headings.
FRAG="$(echo "$VER" | tr '.' '-')"
BLOCK=$(cat <<EOF
## ${DATE} — ${SUB} ${VER}

- [Release notes](${REL}/CHANGELOG.md#${FRAG})

EOF
)

tmp="$(mktemp)"
{ printf '%s' "$BLOCK"; cat "$FILE"; } > "$tmp"
mv "$tmp" "$FILE"
```

After the first real release, **verify** the fragment in the GitHub UI; if the default changelog heading is not `## ${VER}`, adjust the script to match PSR/semantic-release output (for example `## [${VER}]` → different slug).

```bash
chmod +x scripts/prepend-root-changelog.sh
```

- [ ] **Step 3: Commit**

```bash
git add CHANGELOG.md scripts/prepend-root-changelog.sh
git commit -m "docs: root changelog index and prepend helper"
```

### Task 5: Workflow — backend release

**Files:**
- Create: `.github/workflows/release-backend.yml`
- Delete or disable: `.github/workflows/release.yml` (in Task 7)

- [ ] **Step 1: Create `.github/workflows/release-backend.yml`**

```yaml
name: Release (backend)

on:
  push:
    branches: [main]
    paths:
      - "backend/**"
      - ".github/workflows/release-backend.yml"
  workflow_dispatch:

concurrency:
  group: release-backend-${{ github.ref }}
  cancel-in-progress: false

permissions:
  contents: write

jobs:
  semantic-release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0
          persist-credentials: true

      - name: Python Semantic Release
        id: release
        uses: python-semantic-release/python-semantic-release@v10.5.3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          directory: backend
          git_committer_name: github-actions
          git_committer_email: github-actions@users.noreply.github.com

      - name: Prepend root changelog index
        if: success()
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          git config user.name github-actions
          git config user.email github-actions@users.noreply.github.com
          git fetch origin main
          git checkout main
          git pull --rebase origin main
          VER=$(python3 -c "import tomllib; print(tomllib.load(open('backend/pyproject.toml','rb'))['project']['version'])")
          ./scripts/prepend-root-changelog.sh backend "$VER"
          git add CHANGELOG.md
          git diff --staged --quiet || git commit -m "docs: update root changelog index for backend ${VER} [skip ci]"
          git push origin main
```

**Important:** The prepend step must run **only when a new backend version was released**. Do not prepend on every workflow run (that would duplicate index lines). Use the PSR action’s documented step outputs for v10, or capture `project.version` before PSR and compare after PSR; skip prepend when unchanged.

Refine `if:` after reading [python-semantic-release GitHub Action outputs](https://python-semantic-release.readthedocs.io/en/latest/configuration/automatic-releases/github-actions.html) for v10.

### Task 6: Workflow — frontend release

**Files:**
- Create: `.github/workflows/release-frontend.yml`

- [ ] **Step 1: Create `.github/workflows/release-frontend.yml`**

```yaml
name: Release (frontend)

on:
  push:
    branches: [main]
    paths:
      - "frontend/**"
      - ".github/workflows/release-frontend.yml"
  workflow_dispatch:

concurrency:
  group: release-frontend-${{ github.ref }}
  cancel-in-progress: false

permissions:
  contents: write
  issues: write
  pull-requests: write

jobs:
  semantic-release:
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: frontend
    steps:
      - uses: actions/checkout@v6
        with:
          fetch-depth: 0
          persist-credentials: true

      - uses: actions/setup-node@v6
        with:
          node-version-file: ".nvmrc"
          cache: npm
          cache-dependency-path: frontend/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Run semantic-release
        id: sr
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: npx semantic-release

      - name: Prepend root changelog index
        working-directory: .
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          git config user.name github-actions
          git config user.email github-actions@users.noreply.github.com
          git fetch origin main
          git checkout main
          git pull --rebase origin main
          VER=$(node -p "require('./frontend/package.json').version")
          ./scripts/prepend-root-changelog.sh frontend "$VER"
          git add CHANGELOG.md
          git diff --staged --quiet || git commit -m "docs: update root changelog index for frontend ${VER} [skip ci]"
          git push origin main
```

**Refinements required during implementation:**
- Run the prepend step **only if** semantic-release produced a new version (for example `npx semantic-release` exit code 0 **and** `frontend/package.json` version or `frontend/CHANGELOG.md` changed compared to `origin/main` before the run). Skip prepend on no-op runs.
- If `npx semantic-release` leaves the repo in a detached HEAD or on a temp branch, adjust to checkout `main` at the pushed commit (match official semantic-release + Actions patterns).
